# Architecture

A complete map of how `specstory-skill` is built — every file, the data flow,
and the design decisions behind them. This is the canonical structural reference;
`specs.md` (repo root) is the higher-level specification.

## Layered design

```
            command scripts (recap, digest, sessions, search, …)
                              │  parse flags, format output
                              ▼
                         lib/backend.mjs            ← chooses + dispatches
                          /            \
                lib/local.mjs       lib/cloud.mjs   ← two interchangeable backends
                          \            /
                     normalized session shape       ← identical contract
                              │
                         lib/recap.mjs               ← pure parser, fed by either backend
```

The key idea: **both backends return the exact same normalized session object**,
so every command is backend-agnostic. The recap parser is pure (string → object),
so it works identically on local file content and cloud `markdownContent`.

## The normalized session shape

Produced by `local.parseLocalFile()` and `cloud.normalize()`:

```js
{
  source: "local" | "cloud",
  sessionId, projectId, projectName,
  name, userTitle,
  createdAt, updatedAt, startedAt, endedAt,   // ISO 8601 strings
  markdownContent,        // string | null (lazy: only when withContent requested)
  filePath,               // local only; null for cloud
  metadata: {
    clientName, agentName,                    // the tool, e.g. "Claude Code"
    llmModels: [], gitBranches: [], tags: [],
    summary,
  },
}
```

## Backend selection (`backend.mjs`)

`chooseBackend(flags)`:

1. `--cloud` → `cloud`
2. `--local` → `local`
3. auto: local if `.specstory/history` is discoverable (from `--root` or by
   walking up from cwd), else `cloud` if auth exists, else `none` (clear error).

`getSessions(flags, {withContent})` then runs the chosen backend, applies the
`--since/--until` window client-side, sorts newest-first, and caps to `--limit`.

## Local backend (`local.mjs`)

- **Discovery:** `findHistoryUpward(cwd)` (single project) or `findHistoryDirs(root)`
  (recursive, depth-bounded, prunes `node_modules`/`.git`/etc). Each dir that owns
  `.specstory/` is one project; project name = that owning directory's basename.
- **Parsing** the SpecStory Markdown v2.x format:
  - `<!-- <Tool> Session <uuid> (<ts>) -->` → `agentName`/`clientName` + `sessionId`.
  - `_**User (<ts>)**_` / `_**Agent (<model> <ts>)**_` markers → start/end span +
    `llmModels` (synthetic/placeholder tokens filtered).
  - Filename `YYYY-MM-DD_HH-MM-SSZ[-slug]` → `createdAt` + a human title from the slug.
  - Falls back to the first real user line for the name when there's no slug.
- No dependencies; pure Node stdlib (`fs`, `path`, `os`).

## Cloud backend (`cloud.mjs`)

- `resolveToken()`: `SPECSTORY_API_KEY` env → else JWT from
  `~/.specstory/cli/auth.json` (`.cloud_access.token`). The GraphQL endpoint
  accepts the JWT as a Bearer token interchangeably with permanent keys.
- `gql()`: POST, 429 auto-retry, surfaces GraphQL errors.
- `cloudListSessions()` / `cloudSearch()`: see [cloud-api.md](cloud-api.md) for the
  queries and the API quirks they work around (stop-words, loose date filtering,
  the ~100 cap, the `sessions` listing returning 0).

## Recap parser (`lib/recap.mjs`)

`recapFromMarkdown(md, metadata)` → `{ repo, stack, activity, intent, outcome,
summary }`, all fields optional, never throws. Heuristics:

- **repo**: branch (from metadata or stdout), working dir (most-common real path
  prefix, preferring `/Users`/`/home` over container/tmp), owner/name (GitHub URLs
  that match the working-dir basename), recent commits (`git log` output).
- **stack**: languages from file extensions; framework signals (node/python/docker/
  arcpy/…) from marker files.
- **activity**: tool-use counts, files touched, notable non-trivial bash commands.
- **intent**: first user prompt. **outcome**: last user + last substantive agent message.

`renderRecapMarkdown(recap)` renders it as compact markdown for humans/agents.

## Idle-session handling

Terminal agents left open inflate `endedAt` far past the last real activity.
`common.isLikelyIdle(startedAt, endedAt)` flags spans > `IDLE_SESSION_HOURS` (12h);
`recap`/`digest` mark these `⚠️ idle?` so they aren't mistaken for active work time.

## Portability contract

- SKILL.md follows the [Agent Skills spec](https://agentskills.io/specification):
  `name` matches the directory, `description`/`license`/`compatibility` set,
  tool-specific hints live under arbitrary `metadata.*` (ignored by tools that
  don't read them).
- Scripts are referenced by **relative path** (`scripts/…`), never `{baseDir}`.
- Node stdlib only → runs anywhere Node ≥18 exists, no install step.
